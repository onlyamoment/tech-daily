CloakBrowser：从 C++ 源码层处理浏览器指纹

CloakBrowser 是一款经过修改的 Chromium 浏览器，核心目标不是用脚本伪装，而是在源码层修改指纹，让反机器人系统把它识别为普通用户浏览器。

和常见 stealth 方案的区别

市面上大部分方案——如 playwright-stealth、undetected-chromedriver——靠 JavaScript 注入或启动参数调整。每次 Chrome 更新，这些补丁容易失效，而且检测方越来越擅长识别"补丁本身的痕迹"。CloakBrowser 直接在 C++ 层面修改 canvas、WebGL、音频、字体、GPU、屏幕属性、WebRTC、网络时序、硬件报告、自动化信号和 CDP 输入行为等 49 处以上的检测点，然后重新编译成二进制。

因为改的是源码，检测系统看到的就是一个正常的浏览器——它确实是一个正常浏览器，只是指纹被重新编排过。

实测表现

在真实检测服务上的验证结果：

  • reCAPTCHA v3：0.9 分（人类级别，服务器端验证）
  • Cloudflare Turnstile（非交互式）：自动通过
  • FingerprintJS：通过
  • BrowserScan：4/4 全部正常
  • navigator.webdriver：false（源码级 patch）
  • TLS 指纹：与真实 Chrome 一致
使用方式

对 Python 和 JavaScript 用户几乎是零成本替换。接口与 Playwright/Puppeteer 一致，只需改一行 import：

```python

from cloakbrowser import launch

browser = launch()

page = browser.new_page()

page.goto("https://protected-site.com")

```

首次启动自动下载 stealth Chromium 二进制（约 200MB），后续支持后台自动更新。

主要功能
  • **49+ 源码级 C++ 补丁**：覆盖 canvas、WebGL、音频、字体、GPU、屏幕、WebRTC、网络时序、自动化信号、CDP 输入行为
  • **humanize=True**：一键启用类人鼠标曲线、键盘时序和滚动模式，行为检测更难识别
  • **自动更新二进制**：后台检查新版本，始终使用最新 stealth 构建
  • **指纹种子**：`--fingerprint=seed` 固定身份，让同一站点多次访问看起来像老用户；不传则每次启动随机生成新身份
  • **代理与时区联动**：`geoip=True` 根据代理出口 IP 自动匹配时区和语言，避免 UTC+en-US 这种典型机器人信号
  • **持久化 Profile**：`launch_persistent_context()` 让 cookie、localStorage 和缓存跨会话保留，同时绕过隐身模式检测
  • **原生 SOCKS5 支持**：直接支持 `socks5://user:pass@host:port`，无需额外 HTTP CONNECT 处理
部署方式

支持 pip、npm 和 Docker：

```bash

pip install cloakbrowser

npm install cloakbrowser

docker run --rm cloakhq/cloakbrowser cloaktest

```

也支持作为 CDP 服务器长期运行，供外部框架通过 `connect_over_cdp` 连接。

限制与注意
  • 它解决的是"被识别为机器人"的问题，不是 CAPTCHA 求解服务
  • 对 aggressive 站点（DataDome、Kasada）仍需配合住宅代理、headed 模式、完整字体环境和 humanize 行为
  • macOS 版本的指纹覆盖目前不如 Linux/Windows 完整，遇到拦截时可显式切到 Windows 指纹配置
  • Puppeteer 后端会发送更多 CDP 流量，reCAPTCHA Enterprise 偶尔能检测到；Playwright 后端更稳
适合谁

需要做稳定浏览器自动化测试、数据采集、反爬研究或指纹测试的开发者。使用时要遵守目标网站的条款和相关法律边界，不要把它当成绕过限制的万能工具。
